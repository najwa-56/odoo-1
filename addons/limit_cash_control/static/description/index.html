Limit Cash Control in Odoo Point of Sale

Customize Cash Control Mechanism in Point of Sales

About

The problem and solution

Problem 1:
The point of sale module assumes a cashier will always open the day with the previous closing cash and as such sets the opening cash as the previous session closing cash. Although this can always be changed manually, Cashiers typically forget to change the value which eventually leads to accounting issues, especially with the profit/loss statement.
Solution 1:
In some business cases, it will be more rational to be allow the manger set a fixed default opening amount which may be zero or not for each pos.

Problem 2:
By default, Cashiers get to see the session payments values in each account by clicking on the "Close" button, this may not be desired in some organizations that consider them confidential.
Solution 2:
It will be great to selectively choose whether a point os sale session should have access to the closing payment summary or not.

Features

What this module does

Choose whether to set a default opening value or not

Choose whether to hide closing payment summary or not

Configuration

How to set things up

  • After installation, go to the "Limit Cash Control" section of a point of sales settings; if needed, set the default opening cash; and whether to show closing summary or not.
